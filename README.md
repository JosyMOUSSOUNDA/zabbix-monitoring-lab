# Zabbix Monitoring Lab

This project demonstrates the deployment of a monitoring infrastructure using Zabbix to supervise network services and detect incidents.

The objective is to simulate how IT infrastructure monitoring works in a real enterprise environment.

## Project Objectives

- Deploy a Zabbix monitoring server
- Add hosts and monitoring agents
- Configure triggers and alerts
- Create monitoring dashboards

## Lab Environment

The monitoring lab includes:

Zabbix Server  
Monitored Linux hosts  
Virtualized environment using VirtualBox

## Technologies Used

- Zabbix
- Linux Debian
- VirtualBox

## Monitoring Features

The monitoring system tracks:

- CPU usage
- memory usage
- network traffic
- service availability

Triggers were configured to automatically generate alerts when thresholds are exceeded.

## Alert System

Alert notifications are generated when anomalies are detected such as:

- high CPU usage
- service failure
- network downtime
